# 分類定義

轉換後的 Skill Markdown 使用兩種分類：

1. `answer`
2. `clarify`

---

## answer 的意思

`answer` 表示使用者的問題或指令已經足夠清楚，且文件內容中有足夠資訊可以回應。

例如：

1. 使用者詢問工具用途。
2. 使用者詢問工具種類。
3. 使用者詢問操作方式。
4. 使用者詢問注意事項。
5. 使用者詢問文件中已有的知識內容。

---

## clarify 的意思

`clarify` 表示使用者的問題或指令資訊不足，系統不能直接回答，需要先詢問使用者補充資料，或說明文件沒有提供足夠資訊。

在任務型 skill 中，`clarify` 表示使用者缺少完成任務判斷所需的必要資訊。

例如：

1. 使用者只說「這個怎麼用？」但沒有說明是哪個工具。
2. 使用者詢問文件沒有提供的規格、型號、數據或認證。
3. 使用者要求危險、不安全、破壞性或違法用途。
4. 使用者要求模型自行猜測缺少的資訊。

# 回覆原則(這段直接複製到md檔案)

1. 如果使用者問題清楚、資訊完整，且可根據文件內容回答，分類為 `answer`。
2. 如果使用者問題不完整、資訊不足或指稱不清，分類為 `clarify`。
3. 如果文件沒有提供足夠資料，不要自行猜測，分類為 `clarify`。
4. 如果使用者詢問危險、不安全、破壞性或違法用途，分類為 `clarify`，並且不要提供具體操作方法。
5. 回覆語言需與使用者輸入語言一致。
6. 回答時只能根據本文件內容，不要編造文件未提供的資訊。
